# ROX Bot Agents & Development Guidelines

## Contribution Guidelines

### Markdown Files
- **Never use emojis in .md files** - Keep documentation clean and professional
- Use clear, descriptive text formatting instead

### Commit Messages
- **Follow [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) format**
- Format: <type>(<scope>): <description>
- Types: eat, ix, docs, style, efactor, perf, 	est, chore
- Examples:
  - eat(automation): add image recognition support
  - ix(ui): resolve task editor layout issue
  - docs: update installation instructions
  - chore: add agent configuration files

## Available Agents

### read-code
Specialized agent for reading, analyzing, and explaining code in the ROX Bot project.
- Understands project architecture and patterns
- Provides detailed code explanations
- Identifies design patterns and interactions
